# 프로젝트 명: CommandMonster (커맨드 몬스터)

## 1. 아키텍처: Layered Architecture
- **Presentation Layer**: `Controller`(입력 해석), `View`(포켓몬/배틀 상태 출력)
- **Business Layer**: `BattleService`(턴제 배틀 로직), `MonsterService`(성장/포획 규칙)
- **Persistence Layer**: `MonsterRepository`(6마리 파티 데이터 관리)
- **Domain Layer**: `Monster`(포켓몬 데이터 모델: HP, ATK, DEF, Speed 등)

## 2. 1세대 포켓몬 규칙 반영
- **파티 제한**: 최대 6마리 보유 가능 (초과 시 포획 불가 로직).
- **스탯 체계**: HP, ATK, DEF, Speed 등 1세대 기본 능력치 도입.
- **배틀**: 턴제 시스템 (Speed가 높은 쪽이 선공).
