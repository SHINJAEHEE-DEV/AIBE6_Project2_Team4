# 요구사항 명세서 (CommandMonster)

## 1. 핵심 기능 요구사항 (CRUD)

### 1.1 몬스터 포획 (Create)
- [ ] 야생 몬스터 조우 시 이름을 지어주고 파티에 등록할 수 있다.
- [ ] 파티 최대 용량은 **6마리**이며, 초과 시 "파티가 가득 찼습니다" 메시지를 출력한다.
- [ ] 이름은 공백일 수 없으며 최소 1자 이상이어야 한다.

### 1.2 도감 및 파티 조회 (Read)
- [ ] 현재 파티의 요약 목록(이름, Lv, HP상태)을 볼 수 있다.
- [ ] 특정 몬스터의 상세 정보(능력치, 경험치, 포획일)를 확인할 수 있다.

### 1.3 배틀 및 성장 (Update)
- [ ] 야생 몬스터와 1:1 턴제 배틀을 수행할 수 있다.
- [ ] **선공 결정**: Speed 능력치가 높은 몬스터가 먼저 공격한다.
- [ ] **데미지 계산**: `(공격측 ATK * 2) / 방어측 DEF` (단순화된 1세대 공식 적용).
- [ ] 승리 시 레벨이 1 상승하며, ATK/DEF 등 능력치가 1.2배 상승한다.
- [ ] 보유 중인 몬스터의 닉네임을 변경할 수 있다.

### 1.4 방생 (Delete)
- [ ] 파티에서 특정 몬스터를 선택하여 방생(삭제)할 수 있다.
- [ ] 방생 시 파티 목록에서 즉시 제거된다.

## 2. 시스템 및 예외 요구사항
- [ ] 잘못된 입력 시 에러 메시지 출력 후 재입력 유도.
- [ ] 몬스터가 기절(HP=0) 상태일 경우 배틀 참여 불가 처리.
- [ ] 모든 데이터는 메모리(`ArrayList`)에 유지된다.
